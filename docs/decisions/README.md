# decisions/

Журнал принятых решений. Каждое решение — отдельный файл или запись в `log.md`.

**Когда писать:** при выборе технологии, паттерна, архитектурного подхода или продуктового направления — особенно если выбор неочевиден или есть альтернативы.

**Формат записи:**
```
## REF-XXX: Название решения
**Решение:** что выбрали
**Причина:** почему
**Альтернативы:** что отвергли и почему
**Пересмотр:** когда стоит пересмотреть (опционально)
```

## Файлы

- `log.md` — все решения в хронологическом порядке
