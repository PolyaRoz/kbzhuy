# Пользовательские сценарии

_Обновлено: 2026-03-31_

Каждый сценарий — реальная ситуация с конкретным пользователем.
Используй при проектировании фич: если сценарий не покрыт — фича не доделана.

---

## Типы пользователей

| Тип | Описание | Главная боль |
|---|---|---|
| Аня, 28 | Офисная работа, хочет похудеть, готовит мало | Не знает что есть, срывается к вечеру |
| Денис, 34 | IT, уже считает калории вручную, устал | Тратит 40 мин/день на планирование еды |
| Марина, 40 | Семья, двое детей, ограниченный бюджет | Хаотичные закупки, половина портится |
| Олег, 25 | Качалка, набор массы, ест много | Не может разнообразить рацион на 3000 ккал |

## Сценарии

---

### С1. Аня: первый запуск

**Цель:** Понять подходит ли приложение, не потратив много времени.

**Действия:**
1. Скачивает приложение → экран регистрации
2. Email + пароль → аккаунт создан
3. Онбординг шаг 1: рост 165, вес 72, возраст 28, женщина, малоподвижная
4. Шаг 2: цель — похудение
5. Шаг 3: аллергия на орехи, не любит рыбу, бюджет 5000 ₽/нед
6. Шаг 4: завтрак 8:00, обед 13:00, ужин 19:00. Пиво по пятницам
7. Нажимает "Создать план" → ждёт 5 сек → видит план на неделю

**Результат:**
- Аня видит конкретное меню: понедельник-воскресенье, 3 приёма в день
- КБЖУ каждого дня ≈ 1500 ккал (дефицит для похудения)
- Пятница: ужин облегчён, учтено пиво (~400 ккал)
- Нет орехов, нет рыбы
- Список покупок уже сформирован

**Что проверяет:** онбординг, nutri_service, meal_planner, shopping_list

---

### С2. Аня: воскресная готовка

**Цель:** Приготовить еду на неделю за 2-3 часа.

**Действия:**
1. Открывает вкладку "Покупки" → идёт в магазин с телефоном
2. Отмечает чекбоксами что купила (работает офлайн)
3. Дома открывает вкладку "Готовка"
4. Видит пошаговый план: 12 шагов, ~2.5 часа
5. Шаги 3 и 4 — параллельные (курица в духовке + варится гречка)
6. Последний шаг: раскладка по контейнерам
   - 1А: курица + гречка → холодильник (пн обед)
   - 1Б: курица + гречка → морозилка (ср обед, разморозить вт вечером)
   - 2А: творожная запеканка → холодильник (пн завтрак)
   - ...
7. Подписывает контейнеры маркером, ставит по местам

**Результат:**
- 7 дней еды в подписанных контейнерах
- Каждый контейнер привязан к конкретному дню и приёму
- Приложение знает где что лежит

**Что проверяет:** shopping чекбоксы (офлайн), cooking_plan, containers, storage

---

### С3. Денис: обычный рабочий день

**Цель:** Не думать о еде вообще. Открыл → взял → разогрел.

**Действия:**
1. 8:00 — push: "Завтрак: контейнер 2А, холодильник, верхняя полка"
2. Открывает приложение → главный экран → большая карточка: **2А**
   - Содержимое: Творожная запеканка с ягодами
   - Место: Холодильник, верхняя полка
   - Подогрев: Микроволновка 2 мин, 600W
   - КБЖУ: 380 ккал / Б42 Ж12 У28
3. Достаёт контейнер 2А, греет, ест
4. Нажимает "Съел ✓"
5. 12:30 — push: "Обед: контейнер 1А"
6. Открывает → карточка 1А → курица + гречка → разогрел → "Съел ✓"
7. КБЖУ-прогресс обновляется: 980/2100 ккал

**Результат:**
- За весь день Денис принял 0 решений о еде
- Он потратил 0 минут на планирование
- КБЖУ отслеживается автоматически

**Что проверяет:** containers/current, push-уведомления, "Съел", progress

---

### С4. Аня: пятничный срыв

**Цель:** Выпила пиво и съела пиццу, не хочет чувствовать вину.

**Действия:**
1. Пятница, 21:00. Вместо облегчённого ужина — 2 пива и 3 куска пиццы
2. Открывает чат агента → быстрое действие: "Я съел не по плану"
3. Вводит: "2 пива и 3 куска пиццы"
4. Агент оценивает: ~1400 ккал вместо запланированных 350
5. Агент: "Ок, перестраиваю субботу и воскресенье. Убрал перекус в субботу, облегчил ужин в воскресенье. Недельная норма в пределах ±5%."
6. Аня видит обновлённый план на выходные

**Результат:**
- Нет нотации и "ты сорвалась"
- План адаптирован: КБЖУ перераспределены на оставшиеся дни
- Контейнеры на выходные пересмотрены (один перемещён в морозилку на след неделю)
- Аня не чувствует провала

**Что проверяет:** deviation_service, agent/adapt, update_plan, container lifecycle

---

### С5. Марина: контроль сроков годности

**Цель:** Ничего не выбрасывать. Использовать всё вовремя.

**Действия:**
1. Среда, 9:00 — push: "Переложи контейнер 3Б из морозилки в холодильник (обед на четверг)"
2. Марина перемещает контейнер → отмечает в приложении "Готово"
3. Четверг — контейнер 3Б размороженный и готов к обеду
4. Пятница — push: "Контейнер 4А — срок годности истекает завтра"
5. Открывает Хранение → 4А подсвечен красным
6. Три действия: "Съесть сегодня" / "Заморозить" / "Выбросить"
7. Выбирает "Съесть сегодня" → план на вечер обновляется

**Результат:**
- Ничего не испортилось
- Разморозка произошла вовремя
- Истекающий контейнер не потерялся — предложено конкретное действие

**Что проверяет:** prep_tasks, push, storage/expiring, container lifecycle

---

### С6. Олег: набор массы, высокий калораж

**Цель:** Разнообразный рацион на 3000 ккал без одной и той же гречки с курицей.

**Действия:**
1. Онбординг: 185 см, 78 кг, 25 лет, 4 тренировки/нед, цель — набор
2. Nutri-engine: BMR 1850 → TDEE 2870 → +200 профицит = 3070 ккал
3. Макро: Б185 Ж80 У370
4. Генерация плана: агент подбирает разнообразные блюда
   - Не более 2 повторов одного блюда за неделю
   - 5 приёмов в день (завтрак, перекус, обед, перекус, ужин)
5. Олег смотрит план: плов, паста болоньезе, лосось с рисом, бургеры домашние...
6. Список покупок: 25 позиций на неделю

**Результат:**
- 3000+ ккал/день с разнообразием
- Батч-готовка на 5 приёмов × 7 дней = 35 контейнеров
- Контейнеры распределены по морозилке и холодильнику с учётом сроков

**Что проверяет:** nutri_service (высокий калораж), meal_planner (разнообразие), большое количество контейнеров

### С7. Денис: смена цели

**Цель:** Похудел до целевого веса, хочет переключиться на поддержание.

**Действия:**
1. Открывает Профиль → меняет цель: "Поддержание"
2. Вводит новый вес: 82 кг (было 90)
3. Nutri-engine пересчитывает: КБЖУ 2400 → 2100 (без дефицита, maintenance)
4. Агент: "Обновил целевые КБЖУ. Хочешь пересоздать план на эту неделю или доедим текущий?"
5. Денис: "Доедим текущий, с понедельника новый"
6. В понедельник — новый план с обновлёнными КБЖУ

**Результат:**
- Плавный переход между режимами
- Существующие контейнеры не пропадают
- Новый план учитывает новые параметры

**Что проверяет:** profile update, nutri_service recalc, agent dialogue, plan versioning

---

### С8. Марина: бюджетная оптимизация

**Цель:** Уложиться в 4000 ₽/нед на семью из 4 человек.

**Действия:**
1. Онбординг: бюджет 4000 ₽/нед, семья 4 человека
2. Meal planner оптимизирует: сезонные овощи, курица > говядина, крупы
3. Кросс-блюдовая оптимизация: если в понедельник нужна половина качана капусты, вторая идёт в среду
4. Список покупок: "Капуста 1 шт (пн + ср)" — не 2 по полкачана
5. Марина видит итоговую стоимость списка: ~3800 ₽

**Результат:**
- Бюджет соблюдён
- Минимум отходов за счёт перекрёстного использования ингредиентов
- Стоимость видна до похода в магазин

**Что проверяет:** meal_planner (бюджет), shopping_list (агрегация, стоимость)
