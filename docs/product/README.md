# product/

Продуктовая документация: описание фич, пользовательские сценарии, UX-решения.

**Когда читать:** перед разработкой нового экрана или фичи — чтобы понять продуктовый контекст и не строить лишнего.

## Что сюда идёт

- Описания фич с критериями готовности
- User stories и сценарии использования
- UX-решения и обоснования
- Приоритизация

## Источник истины

Полное продуктовое описание — `продуктовое описание.docx` в корне проекта (`../../../продуктовое описание.docx`).
Читать через Python: `zipfile + xml.etree`.
