# Целевая аудитория

_Обновлено: 2026-03-31_

Сегментация по поведению, потребностям и ожиданиям от UX.
Связь с персонажами — см. `user_scenarios.md`.

---

## Сегменты

### S1. Молодые профессионалы (20–30 лет, без семьи)

**Кто:** Офисные работники, IT, фриланс. Живут одни или с партнёром. Доход средний+.

**Персонажи:** Аня (28, похудение), Олег (25, набор массы)

**Поведение:**
- Готовят редко — доставка, кафе, перекусы из магазина
- Импульсные покупки: берут что хочется, без списка
- Пробовали счётчик калорий — бросили через 2–3 недели
- Активно пользуются смартфоном, привычны к приложениям
- Решения принимают быстро, не любят долгие онбординги

**Потребности:**
- Минимум усилий: «сделай за меня»
- Видимый результат через 1–2 недели (вес, энергия)
- Гибкость: ресторан, алкоголь, спонтанные планы — не должны ломать систему
- Разнообразие в рационе — не 7 дней гречки с курицей

**UX-ожидания:**
- Онбординг < 2 минут, результат сразу (план на неделю)
- Один главный экран: «что сейчас есть» — большая карточка контейнера
- Минимум текста, максимум визуала (прогресс-бар КБЖУ, цвета)
- Тёмная тема по умолчанию
- Быстрые действия: «Съел», «Не по плану» — одно нажатие
- Push без агрессии: 2–3 в день, лаконичные

**Ключевой триггер входа:** «Хочу привести себя в форму, но не хочу об этом думать каждый день»

**Риск оттока:** Однообразие рецептов, слишком много push-уведомлений, ощущение «ещё одного счётчика»

### S2. Опытные ЗОЖ-щики (25–40 лет)

**Кто:** Уже считают КБЖУ. Ходят в зал. Знают про макронутриенты, Mifflin-St Jeor, дефицит.

**Персонажи:** Денис (34, IT, считает вручную), Олег (25, набор массы)

**Поведение:**
- Ведут учёт в MyFitnessPal / FatSecret / Excel
- Тратят 20–40 мин/день на планирование + ввод данных
- Часть уже делает батч-готовку, но без системы
- Критичны к точности данных — не доверяют «примерным» расчётам
- Высокие требования к кастомизации (макросплит, timing, refeed)

**Потребности:**
- Автоматизация того, что уже делают вручную
- Точные расчёты — показать формулы, дать возможность переопределить
- Контроль: «я хочу видеть почему такой план, а не просто доверять»
- Экспорт данных, история, графики прогресса
- Поддержка сложных сценариев: рекомпозиция, циклирование калорий

**UX-ожидания:**
- Прозрачность: показать BMR, TDEE, формулу, множитель
- Возможность ручного переопределения КБЖУ-целей
- Детальный view плана: каждый приём, каждый макро
- Dashboard с графиками: вес, калории, белок — по неделям
- Настройки: количество приёмов, распределение макро по приёмам
- API/webhooks для интеграции с фитнес-трекерами (будущее)

**Ключевой триггер входа:** «Я уже всё это делаю, но трачу 3 часа в неделю на планирование — надоело»

**Риск оттока:** Недостаточная гибкость, ощущение потери контроля, «тупой» алгоритм подбора рецептов

### S3. Семьи с детьми (30–45 лет)

**Кто:** 2–4 человека. Один готовит на всех. Бюджет ограничен. Дети едят не всё.

**Персонажи:** Марина (40, двое детей, бюджет)

**Поведение:**
- Закупки хаотичные: 3–4 похода в магазин в неделю
- Готовят каждый день — это рутина, не удовольствие
- 20–30% продуктов выбрасывается (забыли, испортилось, дети не стали есть)
- Планируют на бумажке или в голове — нет системы
- Бюджет — жёсткое ограничение, а не рекомендация

**Потребности:**
- Экономия денег — видеть стоимость до похода в магазин
- Единый план на всю семью с кратным увеличением порций
- Учёт пищевых ограничений членов семьи (аллергии, непереносимости)
- Минимизация отходов: кросс-использование ингредиентов
- Управление сроками годности — не потерять контейнер в морозилке

**UX-ожидания:**
- Стоимость списка покупок — крупно, на первом экране закупок
- Порции: переключатель «на 1 / на 2 / на 4 человека»
- Аллергии и непереносимости — не дополнительная настройка, а часть онбординга
- Напоминания: разморозить, использовать до, переложить
- Простой язык: не «макронутриенты», а «белок, жир, углеводы»
- Возможность шарить список покупок (муж пошёл в магазин)

**Ключевой триггер входа:** «Устала каждый вечер думать что готовить и выбрасывать еду»

**Риск оттока:** Невозможность учесть детей, нет экономии, сложный интерфейс

### S4. Занятые люди с ненормированным графиком (25–45 лет)

**Кто:** Предприниматели, врачи, менеджеры. Много командировок или непредсказуемый график. Деньги есть, времени нет.

**Поведение:**
- Питание хаотичное: кофе утром, бизнес-ланч, ужин дома в 22:00 или ресторан
- Заказывают доставку, но недовольны однообразием и ценой
- Понимают важность питания, но не могут выделить время на систему
- Готовы платить за экономию времени
- Часто пропускают приёмы пищи

**Потребности:**
- Максимальная автономность: план генерируется сам, покупки заказываются
- Адаптация к изменениям в расписании: «сегодня не дома» — план перестроился
- Опция: совместить с доставкой (часть дней — контейнеры, часть — ресторан)
- Быстрый перекус из контейнера в офисе или в дороге

**UX-ожидания:**
- Widget на главном экране телефона: «сейчас ешь контейнер 2Б»
- Минимум тапов: вся ключевая информация на одном экране
- Интеграция с сервисами доставки продуктов (Яндекс Лавка, СберМаркет) — будущее
- Быстрый режим: «у меня 15 минут на обед» — фильтр контейнеров по времени разогрева
- Calendar sync: план питания в Google Calendar

**Ключевой триггер входа:** «Я трачу на доставку 20к/мес и всё равно ем не то — должен быть способ лучше»

**Риск оттока:** Один сбой в плане без адаптации, необходимость ручных действий

### S5. Новички в здоровом питании (18–25 лет)

**Кто:** Студенты, начинающие. Впервые задумались о питании. Не знают что такое КБЖУ.

**Поведение:**
- Питаются фастфудом, полуфабрикатами, лапшой
- Мотивация скачкообразная: начитался/посмотрел видео → «хочу ЗОЖ» → через неделю забыл
- Бюджет минимальный
- Готовить не умеют или делают базовые вещи
- Легко отпугиваются сложными интерфейсами

**Потребности:**
- Образовательный слой: что такое КБЖУ, зачем белок, что такое дефицит — коротко и понятно
- Очень простые рецепты: 5 ингредиентов, 20 минут, невозможно испортить
- Маленькие победы: «ты ел по плану 3 дня подряд» — геймификация
- Минимальный набор оборудования: сковорода, кастрюля, 5 контейнеров

**UX-ожидания:**
- Tooltips и подсказки в интерфейсе (только первые 2 недели)
- Режим «Новичок»: меньше настроек, упрощённый план
- Визуальные награды: стрики, бейджи, прогресс
- Видео-гайд: «Твоя первая батч-готовка за 1.5 часа» (ссылка, не в приложении)
- Копеечные рецепты: гречка, курица, яйца, творог — база

**Ключевой триггер входа:** «Увидел в TikTok как чувак готовит на неделю в контейнерах — хочу так же»

**Риск оттока:** Сложно, непонятно, первая готовка провалилась, нет быстрого результата

## Приоритеты по фазам

| Фаза | Основной сегмент | Почему |
|---|---|---|
| MVP (Фаза 1) | **S1 + S2** | Техническое ядро (КБЖУ, план, контейнеры) ценно обоим. S2 — early adopters, S1 — объём |
| AI-агент (Фаза 2) | **S1 + S4** | Адаптация к срывам (S1) и автономность (S4) — главные ценности агента |
| Хранение (Фаза 3) | **S3** | Контроль сроков, разморозка, минимизация отходов — боль семей |
| Запуск (Фаза 4) | **S5** | Геймификация, onboarding, TikTok-маркетинг — привлечение массовой аудитории |

## UX-выводы для реализации

**Что делать сейчас (MVP):**
- Один главный экран → текущий контейнер, прогресс КБЖУ
- Онбординг за 4 шага, < 2 минут
- Прозрачность расчётов для S2 (показать BMR/TDEE на экране профиля)
- Быстрые действия: «Съел», «Не по плану»

**Что запланировать:**
- Переключатель порций (×1, ×2, ×4) — для S3
- Widget на главный экран — для S4
- Режим «Новичок» с подсказками — для S5
- Стоимость списка покупок — для S3
- Геймификация (стрики, бейджи) — для S5
- Экспорт данных и графики — для S2

**Что НЕ делать:**
- Не делать социальные фичи (ленты, шаринг прогресса) — отвлекает от ядра
- Не делать магазин рецептов / UGC — размывает контроль качества
- Не пытаться быть счётчиком калорий — это антипозиция
